3. Contact Book
Section Purpose
The Contact Book is the central source of truth for all recipient data across the platform.
It is consumed by:
•	Send Message / Campaigns
•	Message Inbox (conversation history)
•	Templates (personalisation and send to list)
•	Email-to-SMS (Send to contact group)
•	Public & private APIs
All data models and actions must be consistent, reusable, and auditable.
________________________________________
3.1 Contact Data Model (Core)
Required Fields
•	Mobile Number (E.164 preferred)
•	Status:
o	Active
o	Opted Out
Default System Fields
These exist for all accounts and are filterable, sortable, and usable in templates:
•	First Name
•	Last Name
•	Email
•	Date of Birth
•	Postcode
•	City / Town
•	Country
•	Created Date
•	Last Updated Date
•	Source (UI / Import / API / Email-to-SMS)
These defaults avoid over-reliance on custom fields for common segmentation.
________________________________________
Custom Fields
Users can create additional fields.
Field definition requires:
•	Field name
•	Field type:
o	Text
o	Number
o	Date
o	Boolean
o	Enum (dropdown)
•	Optional description
Behaviour:
•	Custom fields become:
o	Searchable
o	Filterable
o	Sortable (where appropriate)
o	Available for templates and mail merge
•	Field type determines:
o	Validation
o	Filter operators
o	UI control (date picker, numeric range, etc.)

3.2.4 Search, Filter & Sort
Global Search
•	Searches across:
o	Name
o	Mobile
o	Email
o	Tags
o	Lists
o	All custom fields
•	Partial match
•	Case-insensitive
•	Debounced
Helper text:
“Search across names, numbers, tags, lists, and custom fields”
________________________________________
Filters (Collapsible Panel)
•	Status (Active / Opted Out)
•	Tags (multi-select)
•	Lists
•	Date of Birth (range)
•	Created Date
•	Custom fields (type-aware)

•	Date of Birth

3.2.5 Send Message Behaviour (Key Improvement)
From:
•	Individual contact
•	Multi-select contacts
•	List
•	Dynamic list
•	Tag selection
Action: Send Message
Behaviour:
•	Launches Send Message / Campaign screen
•	Pre-populates:
o	Recipients section with selected entities
•	User continues standard campaign flow
•	Optional confirmation modal:
“You are about to start a campaign to 432 recipients”
