### Security measures implemented

- Feature flag (`BUG_REPORT_WIDGET_ENABLED=false` by default)
- Server-side metadata enrichment (overrides client-sent user/account data)
- `|image` validation rule (checks actual file content, not just extension)
- File extension derived from `getMimeType()` not client filename
- Page URL sanitised (query params stripped to prevent token leakage)
- Console logs sanitised (JWT, Bearer, AWS keys, passwords, secrets, long hex all redacted)
- Webhook HMAC-SHA256 signature verification (always required, no dev bypass)
- Cache-based idempotency on webhooks
- HTTP timeouts (10s configurable) on all external API calls
- GitHub Actions: all untrusted inputs via `env:` blocks (no shell injection)
- Draft PRs only (human review required before merge)
- Rate limit: 5 submissions per 60 minutes per user
