### What this is

A floating bug report widget for the QuickSMS portal (both Customer Portal and Admin Console). Users click a bottom-right button, fill a modal form, optionally attach/annotate a screenshot, and submit. The backend creates a HubSpot ticket. For certain categories, it also creates a GitHub issue that triggers a Claude Code auto-fix pipeline.

### Architecture

```
[Floating Button] → [Bootstrap 5 Modal]
  Form: category, severity, title, description
  Optional: screenshot upload/paste → canvas annotation
  Auto-captured: URL, browser, OS, viewport, user, account
  Console log buffer (last 50, sanitised)

[JS BugReportService]
  Portal → POST /api/bug-report        (customer.auth middleware)
  Admin  → POST /admin/api/bug-report  (AdminAuthenticate middleware)

[BugReportController]
  Validates, sanitises, enriches metadata server-side
  Strips query params from URLs (token safety)
  Derives product area from URL path
  Feature flag gate: abort(404) when disabled

[HubSpotTicketService]
  Creates ticket → uploads screenshot → attaches via notes
  Mock mode when HUBSPOT_ACCESS_TOKEN is empty
  10s timeout, metric logging

[GitHubIssueService]
  Creates issue for portal_bug/ui_layout categories
  Labels: bug-report, auto-fix, portal, ui, priority-high
  Mock mode when GITHUB_TOKEN is empty

[GitHub Actions: bug-auto-fix.yml]
  Triggers on 'auto-fix' label → Claude Code analyses + fixes
  Creates draft PR → webhooks to app

[GitHub Actions: bug-fix-merged.yml]
  Triggers on auto-fix/* branch merge → webhook to app

[BugFixWebhookController]
  HMAC-SHA256 signature verification (always required)
  Cache-based idempotency (1h TTL)
  Updates HubSpot ticket stage + adds timeline note
```

### Security measures

- **Feature flag** — `BUG_REPORT_WIDGET_ENABLED=false` by default. Gates both Blade partial AND controller route
- **Server-side metadata enrichment** — overrides all client-sent user/account data
- **`|image` validation** — checks actual MIME type, not just extension
- **File extension from `getMimeType()`** — not client filename
- **URL sanitisation** — query params stripped, port preserved
- **Console log sanitisation** — 10 regex patterns: JWT, Bearer, AWS keys, passwords, secrets, hex strings
- **Webhook HMAC-SHA256** — always required, no dev bypass
- **Idempotency** — Cache-based duplicate detection on webhooks
- **HTTP timeouts** — 10s configurable on all external API calls
- **GitHub Actions** — all untrusted inputs via `env:` blocks, draft PRs only
- **Rate limit** — 5 submissions per 60 minutes per user
- **Octane-safe** — services registered as `scoped()` in AppServiceProvider
- **CSRF** — webhook route covered by existing `api/webhooks/*` exception in VerifyCsrfToken
- **JS error boundary** — `safeInit()` wrapper hides button if widget fails to initialise
