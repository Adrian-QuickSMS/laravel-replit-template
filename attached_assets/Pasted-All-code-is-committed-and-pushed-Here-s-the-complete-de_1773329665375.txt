## Phase 1 HR Module - Implementation Summary

### 1. Architecture

The HR module is integrated directly into the existing QuickSMS admin console, reusing:
- **Auth**: Existing `customer.auth` session-based middleware
- **Layout**: `layouts.quicksms` with Fillow/Bootstrap 5 theme
- **Tenancy**: Same `tenant_id` scoping via global model scopes
- **Permissions**: QuickSMS owners/admins auto-elevated to HR Admin; HR-specific roles (`employee`, `manager`, `hr_admin`) on `employee_hr_profiles`

### 2. Database Schema (6 tables)

| Table | Purpose |
|---|---|
| `employee_hr_profiles` | Links QuickSMS users to HR identity (role, department, manager, start date) |
| `leave_entitlements` | Per-employee per-year entitlement in quarter-day units |
| `leave_requests` | Full request lifecycle with type, dates, duration, status |
| `leave_audit_log` | Immutable audit trail for all HR actions |
| `company_hr_settings` | Tenant-level defaults (entitlement, notifications, privacy) |
| `bank_holidays` | Reference table for non-working days |

Key design decisions:
- **Quarter-day units** (1 day = 4 units) eliminates floating-point rounding
- `leave_entitlements` has `carried_over_units`, `adjustment_units`, `purchased_units` columns ready for Phase 2
- Unique constraint on `(employee_id, year)` prevents duplicate entitlements

### 3. UI/Page Structure

| Route | View | Purpose |
|---|---|---|
| `/` (dashboard) | Console hub tiles | Messaging + HR module tiles at top |
| `/hr` | HR Dashboard | Summary cards, pending approvals, recent history, upcoming absences |
| `/hr/my-leave` | My Leave | Balance cards, request form, full history with cancel/ICS actions |
| `/hr/admin` | Admin | Tabbed: Approval Queue, Employee List, Add Employee, Settings |

Sidebar navigation updated with HR section (Dashboard, My Leave, Admin).

### 4. Business Logic

- **Pro-rating**: `(remaining_days_in_year / total_days_in_year) * full_entitlement`, rounded UP to nearest quarter day
- **Balance**: Only `status=approved` + `leave_type=annual_leave` deducts. Cancellation restores automatically (no explicit restoration needed since balance is computed dynamically)
- **Sickness/Medical**: Tracked separately, never deducts from annual leave
- **Overlap detection**: Prevents duplicate requests across pending+approved
- **Weekend exclusion**: Working days calculation skips weekends (configurable) and bank holidays
- **Partial days**: Half-day (AM/PM) and quarter-day supported; restricted to single-day requests

### 6. Assumptions

1. QuickSMS account owners/admins are automatically given HR Admin access
2. Holiday year is fixed calendar year (Jan 1 - Dec 31)
3. All employees within a tenant share the same default entitlement
4. Sickness/medical requests can be back-dated (start_date validation is `after_or_equal:today` for now, may need adjustment)
5. Manager can approve requests for anyone in the tenant (simplified for MVP; manager-to-direct-report scoping can be Phase 2)
6. Bank holidays table is empty by default; needs seeding per-tenant or system-wide

### 7. Known Risks & Gaps

| Risk | Mitigation |
|---|---|
| No email sending wired up yet | Template + Mailable ready; just needs `Mail::send()` call in approval flow |
| Bank holidays table empty | Needs seeding from UK GOV API or manual entry |
| No year-end rollover/carry-over logic | Schema supports it (`carried_over_units`); needs cron/command |
| Manager-specific approval scoping | Currently any manager can approve anyone; direct-report filtering available via `manager_id` |
| No duplicate detection for same-day partial leaves | Overlap check is date-range based; two quarter-days on same day would conflict |
